'ibis' Dialect

Types and operations for Ibis dialect The ibis dialect is intended to support porting and eventual open sourcing of an internal hardware development language.

ibis.call (::circt::ibis::CallOp) ¶

Ibis method call

Syntax:

operation ::= `ibis.call` $callee `(` $operands `)` attr-dict `:` functional-type($operands, results)

Dispatch a call to an Ibis method.

Interfaces: CallOpInterface

ibis.class (::circt::ibis::ClassOp) ¶

Ibis class

Syntax:

operation ::= `ibis.class` $sym_name attr-dict-with-keyword $body

Ibis has the notion of a class which can contain methods and member variables.

Traits: HasParent mlir::ModuleOp, IsolatedFromAbove, NoRegionArguments, NoTerminator, SingleBlock, SymbolTable

Interfaces: RegionKindInterface, Symbol

Ibis class instance

operation ::= `ibis.instance` $sym_name `,` $className attr-dict

Instantiates an Ibis class.

ibis.method (::circt::ibis::MethodOp) ¶

Ibis method

Ibis functions are a lot like software functions: a list of named arguments and one unnamed return value.

Can only live inside of classes.

Traits: HasParent, IsolatedFromAbove, SingleBlockImplicitTerminator

Interfaces: FunctionOpInterface, OpAsmOpInterface, RegionKindInterface, Symbol
